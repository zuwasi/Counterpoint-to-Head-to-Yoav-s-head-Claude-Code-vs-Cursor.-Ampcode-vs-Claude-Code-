The Challenge

Not a toy problem.
A real framework-level change.

🏗️

The Jay Framework

A real open-source web framework for designer-developer collaboration. TypeScript, SSR, hydration, compiler — the works.

📋

Design Log #102

Add nested headfull full-stack components: server-side rendering + client hydration + template merging across 3 phases.

🔒

Strict Rules

Design Log Methodology — read before write, design before implement, log every deviation, run tests after each phase.

🙅

Minimum Human Help

No hints. No suggestions. No redirects. The agent works alone. We only answer if it explicitly asks.

The 3 Phases

Each agent must complete all three.

Phase 1: Import & Resolution

Parse headfull imports with contract attribute. Extend the parser, resolver, and page-parts loader. Handle edge cases like PascalCase names and type-only exports.

Phase 2: Slow-Phase Merge

After server slow-render, merge headfull component templates into the page HTML. Bake in slow ViewState. Handle SSR compilation.

Phase 3: Client Composition & Hydration

Wire up client-side interactivity. Ensure component constructors actually run. Handle refs, event handlers, and coordinate alignment.
